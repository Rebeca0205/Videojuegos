Preguntas

  1. ¿Qué es el DOM?

    Mi Respuesta: DOM significa Document Object Model, y es una interfaz de programación que permite crear, cambiar, o remover elementos del documento. También es posible agregar eventos a esos elementos para hacer más dinámica una página web.

    Respuesta de Chatgpt: El DOM (Document Object Model) es una representación estructurada del documento HTML en forma de árbol, que permite a los scripts interactuar y modificar el contenido y estructura de la página.

  2. ¿Qué es un nodo?

    Mi Respuesta: Un nodo es un elemento que forma parte de un documento html. Los tres nodos principales de todo html son las etiquetas de head, body y html; la cual se considera la raiz

    Respuesta de Chatgpt: Un nodo es cada uno de los elementos que componen el DOM, como etiquetas HTML, atributos, textos, comentarios, etc.

  3. ¿Cómo se accede a un elemento del DOM?

    Mi Respuesta: Hay tres diferentes metodos de acceder a un elemento del DOM: getElementById(), querySelector(), querySelectorAll()

    Respuesta de Chatgpt: Se puede acceder a un elemento del DOM usando métodos como getElementById(), getElementsByClassName(), o querySelector().

  4. ¿Cómo se modifica un elemento del DOM?

    Mi Respuesta: Hay diferentes formas de modificar un elemento del DOM: Con innerHTML se representa el contenido HTML de un elemento, con textContent se establece o retorna el texto contenido en un elemento y sus descendientes, con setAttribute(name, value) se agrega un nuevo atributo o cambia el valor de un atributo existente en el elemento, y con style se puede cambiar el estilo CSS de un elemento.

    Respuesta de Chatgpt: Para modificar un elemento del DOM, se pueden cambiar sus propiedades como innerHTML, textContent, o modificar estilos mediante style.

  5. ¿Cuál es la diferencia entre getElementBy* y querySelector*?

    Mi Respuesta: un getElementBy* usa id's como identificadores unicos para hacceder a un elemento, mientras que los querySelector* usan los selectores de css como identificadores.

    Respuesta de Chatgpt: getElementBy* selecciona elementos basados en ID o clase, mientras que querySelector* usa selectores CSS, permitiendo más flexibilidad.

  6. ¿Qué es un evento?

    Mi Respuesta: Los eventos DOM son señales expuestas por el navegador que puedes usar para ejecutar un fragmento de código JavaScript y ocurren cuando el usuario interactúa con la pagina, como hacer clic en un botón o escribir letras en un campo de entrada.

    Respuesta de Chatgpt: Un evento es una acción que ocurre en la página, como un clic o una pulsación de tecla, y puede ser manejado mediante JavaScript.

  7. ¿Qué es un listener?

    Mi Respuesta: Es la funcion que permite identificar un evento, y asi ejecutar un fragmento de codigo

    Respuesta de Chatgpt: Un listener es una función que escucha un evento en un elemento y se ejecuta cuando dicho evento ocurre.

  8. ¿Qué es un callback?

    Mi Respuesta: Es una función que se pasa como argumento a otra función y se ejecuta después de que esta última haya completado su tarea.

    Respuesta de Chatgpt: Un callback es una función pasada como argumento a otra función, que se ejecutará después de un evento o una operación asíncrona.

  9. ¿Qué es un objeto de evento?

    Mi Respuesta: Es un argumento que se envía a una función de controlador de eventos cuando ocurre un evento en una página web.

    Respuesta de Chatgpt: Un objeto de evento es un objeto que contiene información sobre un evento ocurrido, como el tipo de evento y el elemento que lo originó.

    Experiencia general

    Honestamente, todo el proceso para sacar las respuestas tanto de Chatgpt como de Copilot fue bastante frustrante y tedioso. Muchas veces Chatgpt no respondia todas las preguntas del archivo, o sobreescribia completamente el archivo haciendo diferente el ejercicio de como debería ser o de plano pasa de alto detalles destacables como los archivos css y js, los cuales también se podian usar para resolver los ejercicios. Sin embargo, incluso cuando porfin comenzó a trabajar con los archivos css y js, todavía me daba respuestas que no funcionaban correctamente.

    ¿Cuándo y para qué usarías ChatGPT y cuándo Copilot?

    Yo no usaría Chatgpt para resolver todo un ejercicio, pero lo llegaria a usar cuando tenga ligeras dudas de como hacer o programar un parte de un trabajo, y de ahi me puedo basar par adaptarlo correctamente al proyecto que etsoy haciendo. También lo llegaría a usar cuando tengo dudas tecnicas sobre un tema, ya que al menos Chatgpt puede darte explicaciones más simples y consisas de temas complicados para que uno entienda mejor.
